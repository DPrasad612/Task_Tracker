====================================================
CORE TASK DATE SYSTEM
=====================

Replace the current strictly weekly-task behavior with a fully dynamic date-driven tracking system.

Tasks should no longer exist only inside a fixed weekly structure.

Instead:

* every task should operate based on:

  * start date
  * optional end date
  * active lifecycle state

====================================================
TASK CREATION FLOW
==================

When creating a new task:

1. Ask for:

   * task title
   * category
   * priority
   * optional subtasks

2. Add:

   * Start Date (required)
   * End Date (optional but recommended)

3. Add a toggle:
   “Enable time-based task”

4. If enabled:

   * allow user to specify:

     * time
     * optional note/reminder text
   * display this below the task name in the tracker UI

5. Tasks should NOT appear immediately if the start date is in the future.

====================================================
UPCOMING TASK SYSTEM
====================

If:
current date < task start date

Then:

* the task must appear inside a separate:
  “Upcoming Tasks” section

Requirements:

* upcoming tasks must NOT appear inside the active tracker
* upcoming tasks must be visually separated
* upcoming tasks should display:

  * title
  * start date
  * optional end date
  * optional scheduled time
* keep the existing pastel SaaS aesthetic

====================================================
AUTOMATIC TASK ACTIVATION
=========================

When the current date reaches the task start date:

* automatically move the task into the active tracker
* no manual refresh
* no reload button
* no user action required

Requirements:

* use reactive date-based filtering
* transition should feel seamless
* preserve:

  * ordering
  * subtasks
  * progress state
  * analytics state

====================================================
ACTIVE TASK RULES
=================

Only active tasks should appear in the main tracker.

A task is ACTIVE when:
current date >= start date
AND
current date <= end date (if end date exists)

Requirements:

* active tasks always appear before upcoming or expired tasks
* preserve user ordering
* keep tracker responsive and lightweight

====================================================
EXPIRED TASK RULES
==================

When:
current date > end date

Then:

* automatically remove the task from the active tracker
* prevent further completion interaction
* prevent expired tasks from affecting:

  * analytics
  * streaks
  * dashboard metrics
  * active progress

Expired tasks should:

* never remain in the active tracker
* optionally become archived internally for future history support
